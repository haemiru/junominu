## 1. 플랫폼 팩트 (기준일: **2026-07-22**)

전략은 여기서 파생된다. 이 절이 낡으면 아래 전부 재검토(§10).

### 알고리즘

| 항목 | 사실 |
|---|---|
| 피드 구조 | **For You = 알고리즘 추천** / **Following = 역순 시간순(랭킹 없음)** |
| 초기 노출 | 팔로워의 **약 2~5%** 에 먼저 뿌리고, 반응 보고 증폭 |
| 최강 신호 | **답글**. Mosseri: "한 계정의 **답글 총합은 게시물 총합만큼 가치 있다**" |
| 골든타임 | **60~90분** (기존 30~60분에서 상향) |
| 랭킹 신호 5종 | 좋아요 확률 · **답글 참여** · 팔로우 확률 · **프로필 클릭 예측** · 스크롤패스 확률 |
| 팔로워 수 | 도달에 **거의 영향 없음**. 초기 답글 속도가 지배 |
| 억제 | 노골적 engagement bait(=「좋아요 눌러주세요」류) 가시성 축소 |
| 개인화 | "Dear Algo" — 이용자가 추천을 직접 조정 |

> **프로필 클릭이 랭킹 신호**라는 게 중요하다. 글이 "이 사람 누구지?"를 만들면 알고리즘이 한 번 더 밀어준다. 곧 **프로필(=랜딩) 정비가 도달에도 이득**이다.

### 기능 (2025 하반기~2026 신규)

- **장문 텍스트 첨부 — 최대 10,000자.** ⚠️ 본문 한도는 **여전히 500자**. 500자 본문 + "text attachment"로 장문을 붙이는 구조(모바일 전체화면 / 웹 전용창에서 열림). 서식 제한적, 중간 이미지 삽입 불가. **무료**(X는 유료 구독 25,000자).
- **답글 예약 발행** — 연결 스레드의 뒷편을 미리 예약 가능.
- **인사이트 강화** — **"View sources"(유입 출처)** + 인구통계 확대. 단 **링크 클릭·프로필 방문은 표준 지표로 확인 안 됨**(§7 참조).
- **DM 이미지 전송** 지원.
- **크리에이터 허브** 개설.
- **인스타 스토리로 크로스 공유** (계정 연동 시).
- **광고: 미국·일본만.** 한국은 아직 → **유료 증폭 옵션 없음, 100% 오가닉.**

### 링크 (⚠️ 과거 기준 정정)

- 2024년엔 링크 글이 **눈에 띄게 눌렸다**. → **2025~2026에 리밸런싱됨.** Mosseri가 URL 포함 글에 **더 높은 값**을 주도록 랭킹을 조정했다고 확인(한 달 이상 적용).
- **현재 규칙: 「링크 금지」가 아니라 「맥락 없는 링크 금지」.** 그냥 링크만 던지면 여전히 저조. 링크를 **생각·질문에 싸서** 주면 불이익 없음.
- 그래도 안전판: 앵커 같은 중요 글은 **링크를 마지막 편**에 둔다(완독률 보호 목적이지, 페널티 회피 목적이 아님).

### 한국 시장 (⚠️ 전략 판단이 바뀌는 지점)

| 항목 | 수치 |
|---|---|
| 국내 MAU | **약 692만** — X(757만) 턱밑 추격 |
| 국내 이용시간 | 전년 대비 **+80%** |
| 글로벌 | MAU 5억 돌파(2026-06), DAU 1.43억으로 X 추월(2026-01) |
| **국내 연령** | **1020 잘파세대 57%**, 그중 **20대가 40%** |

**→ 결론 (⚠️ 2026-07-24 사용자 지시로 전면 갱신 — 이전 결론 철회).**

스레드 한국 코어가 20대인 것은 **사실로 유지**한다. 하지만 하성재님이 스레드를 하는 **목적**은 '메이커로서의 인지도(동료 개발자·지망생 대상)'가 **아니라**, 내가 만든 앱·서비스의 **실사용자에게 "당신에게 꼭 필요한 걸 만들어주는 사람"으로 각인**되는 것이다(`SKILL.md` §2 ★).

> **스레드의 1차 임무 = 「사용자·대중에게 '진짜 쓸모를 만드는 사람'으로 인지」.** 콘텐츠는 철저히 사용자 관점(서비스 스토리, `SKILL.md` §2 ★). 동료 개발자를 겨냥한 기술 자랑·바이브코딩 강의 톤이 아니다.
> ⚠️ 플랫폼 현실은 잊지 말 것: 특정 앱의 **직접 사용자 획득**(예: 30~40대 공인중개사·부모)은 스레드보다 **맘카페·업계 커뮤니티**가 효율적이다. 스레드는 그걸 대체하는 게 아니라, **넓은 신뢰·평판("저 사람은 진짜 쓸모를 만든다")**을 쌓아 그 결로 사용자·잠재고객에 닿게 한다.

## 2. X 1인 기업가 플레이북 → 스레드 이식표

X 솔로프러너를 굴리는 엔진은 네 개다. 대부분은 겉모습(지표 자랑·권위 톤)을 베끼다 죽는다.

| X의 엔진 | 스레드 이식 | 비고 |
|---|---|---|
| **Build in public** — 제품이 아니라 **여정**을 판다 | ◎ **더 잘 먹힘** | 스레드는 "과정·감정"에 후하다 |
| **답글이 성장의 8할** | ◎ **X보다 더 셈** | Mosseri가 명시. 최우선 투자처(§5) |
| **고정글 = 랜딩페이지** | ○ 그대로 | 프로필 클릭이 랭킹 신호라 이득이 두 배(§3) |
| **아웃라이어 재활용** | ◎ **더 유리** | 추천 기반이라 재탕이 매번 **새 사람**에게 닿음(§8) |

### ✗ 옮기면 죽는 것

- **허슬브로 톤 · MRR 스크린샷 자랑** → 한국 스레드에선 즉사 (`SKILL.md` §2 금지 항목과 동일)
- **"전문가/권위" 포지션** → 스레드는 **동료·이웃 톤**이 이김
- **팔로워 수를 KPI로** → 도달이 팔로워와 거의 무관. 팔로워는 *결과*지 *목표*가 아님(§7)
- **하루 여러 개 발행** → 새 글이 이전 글의 답글 누적을 잡아먹음(§4)
- **영어권 계정 벤치마킹** → 한국 스레드는 감정·일상·공감이 훨씬 셈

### ◎ 스레드가 오히려 유리한 점 (하성재님 한정)

1. **팔로워 0에서도 터진다.** 알고리즘이 팔로워 그래프를 거의 안 봄 → 후발 진입 페널티가 X보다 작다.
2. **따뜻한 존댓말 구어체 = 네이티브 톤.** X 솔로프러너들이 억지로 만들려는 "authentic"을 이미 갖고 있다.
3. **재고가 압도적.** live 5개(짱샘의 책방·킁킁메이트·i-talk·아이복지모아·트랜스크립토) + building 3개(소리야 놀자!·중개프로·매일 두뇌 활동). **building 3개 = 실시간 연재 소재**가 계속 나온다.
4. **20대 코어 = 코칭·외주 구매층.** "22년 IT 경력자가 AI로 혼자 만든다"는 서사는 이 층에 강하게 꽂힌다.

### 2-B. 메타 상품화 — 제작 과정 자체를 판다

`winning-patterns.md` 분해 #1의 핵심 이식점. 이기는 계정은 **결과물만 파는 게 아니라 "그걸 만든 16시간의 과정"을 상품으로 만든다**(실제 지시 원문·전체 기록·셋팅 가이드 = 72페이지). 하성재님에게 이게 특히 잘 맞는 이유:

- 이 채널의 축이 이미 **🔧 빌드 로그·💡 how-to·`detail.prompts`**다. 즉 **팔 물건(과정 기록)이 매일 쌓이고 있다.**
- 관문(강의·코칭·외주)과 직결: 스레드에서 **과정을 공개**하고 → 더 깊은 **전 과정·프롬프트·템플릿**을 강의/코칭/유료 문서로 → `/contact`.
- 순서가 중요: **회고·소통이 판매보다 먼저**다(분해 #1 관찰). 과정을 아낌없이 공개해 신뢰를 쌓고, 판매는 **맥락에 싸서** 뒤에 붙인다. 대놓고 팔면 스레드에선 죽는다(§2 허슬 금지).
- **역위험 보장(선택)**: 코칭·강의에 "안 맞으면 환불" 정도의 약한 보장은 신뢰를 높인다. 단 윤자동식 "못 벌면 100% 환급" 같은 **수익 약속형은 피한다**(과장·기대관리 리스크, §1 금지 톤).

## 3. 프로필 = 랜딩페이지

프로필 클릭이 **랭킹 신호**이자 **퍼널 입구**다. 두 배로 중요.

**3층 구조로 설계한다.**

1. **한 줄 소개** — "무엇을 하는 사람인가"가 3초에 잡혀야 한다.
   - 원칙: **정체성 + 증거 + 대상**.
   - ✅ **적용 완료 (2026-07-22) — 현재 bio:**
     ```
     IT 회사 22년, 정작 제 손으로 만든 제품은 하나도 없었어요
     지금은 바이브코딩으로 혼자 기획·개발·배포합니다
     7개월 · 서비스 8개 · 5개 운영 중
     막힌 지점까지 다 공유합니다 ↓
     ```
     - 1행 = 반전형 훅(정체성·낙차) / 3행 = 증거 / 4행 = 대상 예고(🔧 빌드 로그 축을 약속).
     - 이전 bio의 "1:1 코칭·외주 문의 ↓"는 **뺐다** — 처음 보는 사람에게 바로 "문의하세요"는 전환이 안 난다. 판매는 **고정글과 사이트(`/contact`)** 가 받는다.
   - ⚠️ **드리프트 주의**: `7개월`·`서비스 8개`·`5개 운영 중`은 시간과 `projects.js`에 따라 낡는다. **월 회고 때 같이 확인**하고, 프로젝트를 추가하면 bio 숫자도 고친다.
2. **고정글(핀)** — 신규 방문자가 반드시 보는 단 하나.
   - **1순위 = 계정 소개 앵커**("나는 누구인가"). 프로젝트 하나가 아니라 **사람**을 소개하는 글이어야 프로필→고정글이 한 메시지로 이어진다.
   - 2순위 = 성적이 가장 좋은 **프로젝트 앵커 제작기**(계정 소개 앵커가 아직 없을 때).
   - ✅ **적용 완료 (2026-07-22)**: 계정 소개 앵커 4편 고정 중 — bio와 **같은 반전형 서사**("IT 22년, 제 손으로 만든 서비스는 없었다" → 바이브코딩으로 5개 운영). 로그는 `threads-log.md` 참조.
   - 교체 기준: 더 나은 **계정 소개** 글이 나왔을 때. 프로젝트 앵커가 아무리 터져도 고정글은 계정 소개가 지킨다(프로젝트 앵커는 본문에서 링크로 연결).
3. **링크** — `junominu.com` 하나만. (사이트가 이미 프로젝트·블로그·`/contact` 허브 역할)

## 4. 콘텐츠 축과 주간 리듬

### 발행 빈도 (⚠️ 기존 "하루 1~2개"에서 하향)

- **하루 1개 상한. 주 3~5개 권장.**
- 이유: 하루에 여러 개를 올리면 **새 글이 이전 글의 답글 누적을 끊는다.** 스레드는 답글이 시간을 두고 쌓여야 증폭된다.
- 하한: 주 2개 미만이면 모멘텀이 안 붙는다.
- ※ 소스마다 "하루 1회" ~ "주 2~5회"로 갈린다. **주 3~5개 + 하루 1개 상한**이 두 권고의 교집합이자 안전한 출발점. 3개월 데이터가 쌓이면 §7 기준으로 직접 조정.

### 축 4개 (`SKILL.md` §4 로테이션을 채널 관점으로 재배분)

| 축 | 비중 | 무엇 | 관문 연결 |
|---|---|---|---|
| 🔧 **빌드 로그** (신규·핵심 엔진) | 35% | 오늘 뭘 만들었나·막혔나·고쳤나. building 3개가 소재 | 신뢰 → 코칭·외주 |
| 🫶 **문제·공감 사연** (시그니처) | 25% | 한 장면 훅 → 공감 → 질문 | 따뜻함 → 브랜드 |
| 💡 **바이브코딩 인사이트·how-to** | 25% | "비개발자가 ○○ 붙인 법" (`detail.prompts`) | **직접 → 코칭·강의** |
| 📈 **마일스톤·지표** | 15% | "7개월, 혼자, 서비스 8개" | 증거 |

**앵커 제작기**(`SKILL.md` §3)는 이 위에 얹는 **월 1회 이벤트**. 프로젝트당 딱 하나, 고정글 후보.

### 토픽 기본값 (⚠️ 2026-07-24 재변경 — 사용자 관점)

- **기본 = 사용자·도메인 토픽.** 중개프로면 #부동산, 아이복지모아면 #육아·#발달장애 등 — **실사용자의 세계**에서 고른다(§1 결론·`SKILL.md` §2 ★ 목적).

## 5. 답글 전략 ← **가장 큰 구멍이었던 것**

X 플레이북의 8할이고, 스레드에선 **더** 세다. Mosseri: *"답글 총합 ≈ 게시물 총합의 가치."*
**글을 못 쓰는 날은 있어도, 답글을 안 다는 날은 없어야 한다.**

### 왜 되는가

답글은 (a) 상대 팔로워에게 노출되고 (b) 내 계정 참여 점수를 올리고 (c) **프로필 클릭**(랭킹 신호)을 직접 만든다. 팔로워 0이어도 **남의 관중석을 빌릴 수 있다.**

### 어디에 다나 — 3개 링

1. **동료 링 (매일, 60%)** — 비슷한 규모의 1인 메이커·바이브코딩 계정. 여기서 진짜 관계가 생기고, 상호 답글이 초기 골든타임을 살린다. **20~30개 계정을 리스트로 유지.**
2. **큰 계정 링 (매일, 30%)** — 팔로워 많은 메이커·IT·창업 계정의 **갓 올라온 글**. 늦게 달면 안 보인다. **업로드 30분 내**가 승부.
3. **타겟 링 (주 2~3회, 10%)** — 발달장애 육아·돌봄·중개사 관련 글. 제품 사용자 접점. 파는 게 아니라 **공감만** 남긴다.

### 답글 원칙

- **"좋은 글이네요" 금지.** 한 줄 감상은 아무 신호도 못 만든다.
- 이기는 답글 = **① 내 경험 한 조각 + ② 상대에게 되묻는 질문.** 대화가 이어지면 그게 두 번째 노출이다.
- **홍보 금지.** 답글에 링크 붙이지 않는다. 프로필 클릭이 알아서 일한다.
- 내 글에 달린 답글엔 **전부** 답한다(골든타임 60~90분). 이게 알고리즘 연료.
- 답글도 **예약 발행 가능** → 연결 스레드 뒷편은 미리 걸어두고, 사람 상대는 실시간으로.

### 답글 크래프트 — 내 글에 달린 답글에 "어떻게" 답하나 (분해 #1의 진짜 엔진)

`winning-patterns.md` 분해 #1에서 도달을 만든 건 본문이 아니라 **답글**이었다. 이기는 답글엔 공식이 있다 — 톤은 하성재님 것(따뜻한 존댓말)으로 바꾸되 뼈대는 그대로 이식한다.

- **되받되, 방어하지 말고 더 솔직하게.** 의심·시비는 최고의 소재다. 반박하지 말고 **증거 한 조각 + 인정**으로 되받는다.
  - 예) "이거 AI가 쓴 거 아녜요?" → *"맞아요, 초안은 AI가 씁니다 ㅎㅎ 근데 뭘 넣을지 고르고 틀린 거 잡는 건 아직 제 몫이더라고요. 그 부분이 제일 오래 걸려요."*
  - 예) "카피 아니냐" → *"의심하실 만해요. 근데 막힌 지점이랑 커밋까지 시간순으로 다 공개돼 있어서, 지어내려면 그게 더 품이 들어요 😅"*
- **끝을 되물음으로 닫는다.** 감상("감사합니다")으로 끝내면 대화가 죽는다. **질문 하나를 되던져** 두 번째 노출을 만든다.
- **빈틈·실패를 먼저 인정.** "저도 그거 세 번 갈아엎었어요" 류가 권위 세우기보다 훨씬 세게 붙는다(§2 동료·이웃 톤).
- **가벼운 위트 OK, 과장 금지.** 이모지 1개·"ㅎㅎ" 정도. 돈·성과 자랑으로 받아치지 않는다(§1 금지).
- **일관된 목소리.** 답글도 본문과 같은 사람이어야 한다 — 따뜻하고, 솔직하고, 작아도 진짜인 톤.
- ⚠️ **복제 금지**: 윤자동은 답글을 **AI가 페르소나로 자동 응답**했다. 하성재는 **골든타임에 사람으로 직접** 단다 — 그 진심이 이 채널의 차별점이라 자동화하지 않는다.

## 7. KPI와 계측

### 보지 말 것 / 볼 것

- ❌ **팔로워 수를 성과로 보지 말 것** — 도달과 거의 무관. 다만 **계측 해금 관문**이라는 실용적 의미는 있다(아래).
- ✅ **답글 수 / 조회수** — 알고리즘이 실제로 보는 것
- ✅ **프로필 클릭** — 랭킹 신호이자 퍼널 입구
- ✅ **최종: junominu.com 유입 → `/contact` 폼 제출** ← 여기가 진짜 목표

### ⚠️ 스레드 인사이트는 팔로워 100명부터 (2026-07-22 확인)

계정 인사이트 탭이 **"인사이트 대기 중 — 팔로워가 100명이 되면 다시 방문하세요"** 로 잠겨 있다. 그래서:

| 지표 | 지금(100명 미만) | 100명 이후 |
|---|---|---|
| **글 단위 조회·좋아요·답글** | ✅ **각 글에서 볼 수 있다** — 계정 인사이트가 잠겨도 글별 수치는 살아 있다 | ✅ |
| 계정 요약·"View sources"(유입 surface) | ❌ 잠김 | ✅ |
| 팔로워 인구통계 | ❌ 잠김(민감정보라 가장 늦게 열림) | ✅ |
| **사이트 유입·문의** | ✅ **Vercel Analytics로 이미 측정 중** — 스레드와 무관하게 잡힌다 | ✅ |

- **→ 지금 당장의 계측은 「글 단위 수치(수동 기록)」 + 「Vercel Analytics」 조합으로 충분하다.** 월 회고 때 상위 글의 조회·답글을 눈으로 읽어 `threads-log.md`에 옮기면 아웃라이어 판정(§8)이 가능하다.
- **팔로워 100 = 유일하게 정당한 팔로워 목표.** 허영 지표가 아니라 **계측을 여는 열쇠**라서다. 이것만 넘기면 다시 팔로워는 안 본다.
- 💡 **우회로(검증 필요)**: **비즈니스 계정으로 전환하면 100명 관문이 적용되지 않는다**는 보고가 있다(서드파티 출처 — Meta 공식 문서로 확인되지 않음). 실제로 코칭·외주를 파는 계정이라 전환 자체가 부자연스럽지도 않다. **전환 전에 다른 제약이 붙는지 확인할 것.**

### ✅ 계측 현황 (2026-07-22 구축 완료)

- **Vercel Analytics 가동** — 스레드 유입은 Referrers의 `l.threads.com` 으로 잡힌다.
- **UTM** — 링크에 `?utm_source=threads&utm_campaign=<글슬러그>` 를 붙이면 대시보드 **UTM Parameters** 탭에서 **글 단위 귀속**.
- **문의 귀속** — `src/attribution.js` 가 첫 진입 때 utm/referrer를 잡아두고, `/contact` 의 Tally 폼에 숨김필드 **`src`** 로 실어 보낸다 → "어느 글이 문의를 만들었나"가 응답에 기록된다.
- ⚠️ 읽을 때: **본인 방문이 섞이고**(자동 제외 없음) **광고 차단기 방문자는 누락**된다 → 총 Visitors가 아니라 **referrer/UTM이 붙은 건**만 센다.

## 8. 아웃라이어 재활용

스레드는 추천 기반이라 **같은 이야기를 다시 써도 매번 새 사람에게 닿는다.** X보다 유리한 지점이니 반드시 쓴다.

1. 월 1회 회고에서 **조회수·답글 상위 2개**를 아웃라이어로 지정.
2. 왜 터졌는지 한 줄로 적는다 — **훅 공식? 소재? 감정?** (`SKILL.md` §5 공식명으로)
3. **2~4주 뒤 각도만 바꿔 다시 쓴다.** 같은 소재 + 다른 훅 공식, 또는 같은 훅 공식 + 다른 프로젝트.
4. 3회 이상 이긴 공식은 `SKILL.md` §5 훅 라이브러리에 **하성재님 검증 예시로 승격**.
5. 반대로 2회 연속 폭망한 각도는 로그에 **"버림"** 표시.

> 새 아이디어를 짜내는 것보다 **이긴 아이디어를 반복**하는 게 항상 싸다.
