JUNOMINU · GROWTH ROADMAP

바이브 코딩 사이트로
키워가는 계획

junominu.com을 "결과물로 보내주는 브릿지"에서, 혼자 AI로 어떻게 만들어 가는지(과정)를 보여주는 사이트로 키우기 위한 작업 문서입니다.

마지막 업데이트: 2026-06-02  ·  관리 기준: ROADMAP.md (이 화면은 보기용 ROADMAP.html — 항상 같은 내용)

한 줄 방향  → 지금은 "이거 만들었어요(링크)"에서 끝납니다. 앞으로는 "이렇게 만들었어요(과정·삽질·배운 점)"를 보여주는 쪽으로 무게중심을 옮깁니다. 그게 바이브 코딩 사이트의 진짜 차별점이에요.

큰 그림 — 3단계 (쉬운 것부터)

1
지금 페이지 안에서 키우기
코드 구조를 바꾸지 않고(라우터 없이) 한 페이지에 섹션만 추가 — About · 지표 · Now. 가장 빠르게 효과가 보입니다.
2
프로젝트 상세 페이지 만들기
페이지 이동(라우터)을 도입해, 카드를 누르면 "이 프로젝트를 이렇게 만들었다"는 상세 이야기가 나오게. 사이트의 핵심 콘텐츠입니다.
3
블로그 시작
글을 쓰면 검색으로 사람들이 들어오고, 신뢰가 쌓입니다. 글은 쌓일수록 복리로 작용해요.

해야 할 일 — 콘텐츠 5영역

1. 메이킹 스토리 / 빌드 로그 가장 중요

카드는 "만들었어요"에서 끝납니다. 여기에 "이렇게 만들었어요(스택·기간·막혔던 점·핵심 프롬프트·회고)"를 더하면, 이 사이트만의 색이 생깁니다.

  • ✓프로젝트 데이터에 상세 필드 추가 (스크린샷·스택·기간·막힌 점·프롬프트·회고)
  • ✓카드 클릭 → 외부로 바로 보내지 말고 내부 상세 페이지 경유 → 거기서 "사이트 열기" (Phase 2)
  • ✓"0 → 배포까지 N일" 타임라인 표기 (Phase 2)
  • ✓상세 페이지 라우팅 2단계

2. Build in Public — 살아있는 사이트

"성장하는 작업실" 느낌을 주는 요소들. 지금 뭘 하는지, 얼마나 쌓였는지가 보이면 사이트가 살아있게 느껴집니다.

  • ✓Now 섹션 — "지금 만들고 있는 것" (주/월 단위로 갱신)
  • ✓지표 카운터 — 프로젝트 수 / 운영 중 / 작업 중 / 개월째 (자동 계산)
  • ✓타임라인 — 첫 프로젝트부터 시간순 기록(저니) — 홈 JOURNEY 섹션

3. 블로그

검색 유입과 신뢰의 핵심. "이 사람 진짜 한다"를 증명하고, 글 한 편이 계속 사람을 데려옵니다.

  • ✓마크다운 글 파이프라인 구성 (가볍게 시작) — blogData.js + marked
  • 글감: 바이브 코딩 워크플로우, 도구 리뷰(Claude Code·Cursor·Vercel·Supabase), 삽질기
  • ✓글 목록 + 개별 글 페이지 3단계
  • ✓기본 SEO (메타·OG·사이트맵) — OG 전용 이미지만 남음

4. About / 나

누구인지, 왜 바이브 코딩을 하는지(비개발자 → 1인 메이커 서사). 이미 전자책까지 있으니 그 이야기와 강하게 연결됩니다.

  • ✓About 섹션 — 소개 + "왜 바이브 코딩을 하는가" 서사 (초안)
  • ✓연락처 / SNS / GitHub 링크 — GitHub · 짱샘의 책방 · Email
  • ✓전자책 "바이브 코딩으로 1인 사업가 되기" 서사와 연결 — About에 문장 추가(공개 URL 나오면 링크)

5. 부가 (여력 될 때)

팔로워를 모으고, 사람들이 다시 찾게 만드는 장치들.

  • ✓내 스택/도구함 — 홈 STACK 섹션
  • ✓프롬프트 공유 — /prompts 프롬프트 노트(전 프로젝트 detail.prompts 자동 집계). 보일러플레이트/템플릿은 아직
  • 뉴스레터 구독 폼 — ⛔ 도구 선택 + 외부 계정 필요(사용자 결정 대기)
